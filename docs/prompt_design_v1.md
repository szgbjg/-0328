# Prompt 设计决策文档 v1.0

## 1. Question Creator（问题生成器）

### 输出格式决策 [手动填写]
- 输出：JSON 数组
- 示例：["问题1", "问题2", "问题3"]
- 数量控制：3个左右（最少1个，最多5个）

### 质量检查清单 [手动设计]
- [ ] 问题是否依赖外部知识？
- [ ] 是否出现"根据上文/上述"等指代词？
- [ ] 问题之间交集是否最小？
- [ ] 是否可被context直接回答？

### 特殊处理 [手动设计]
- context不足时的降级策略：仅提取极少信息的1个问题，并确保以单元素数组 ["问题1"] 返回。如果完全无信息则返回空数组 []
- 问题重复检测方法：采用聚类自检法：若两道题答案高度一致或提取的依据重叠度高，则自动舍弃或合并其中一题。

## 2. Facet Planner（角度规划者）

### 输出格式决策 [手动填写]
- 输出：JSON 数组
- 示例：["技术实现", "应用落地", "风险治理"]
- 数量：1-8个（根据复杂度）

### 角度命名规范 [手动设计]
- 允许：名词或短词组（2-4字最佳）
- 禁止：长句、反问句、疑问词
- 示例：✅"临床安全" ❌"从临床安全角度怎么看"

---

## 3. Facet Reducer（角度筛选器）[关键角色]

### 输入 [手动明确]
- query: string
- facets: array（长度&gt;8）

### 输出 [手动明确]
- 严格8个facets的JSON数组
- 不得新增、改写、合并

### 筛选策略优先级 [手动设计]
1. 覆盖面广 &gt; 覆盖面窄
2. 高价值维度 &gt; 边缘角度
3. 通用表达 &gt; 具体表达

---

## 4. Facet Expander（角度补充器）

### 输入 [手动明确]
- query: string  
- existing_facets: array（长度2-8）

### 输出 [手动明确]
- 仅新增facets的JSON数组（0-6个）
- 不得重复已有角度（包括近义）

### 补充策略 [手动设计]
- 从哪些维度补充？根据临床场景穷举：如缺漏，优先补充"临床获益"、"不良反应/风险"、"用药禁忌"、"患者生活建议"、"相互作用"等。
- 如何判断角度是否已覆盖？要求 LLM 将生成的补充 facet 与原 facets 提取核心名词进行语义比对，若属于同义词/近义词（如"临床安全"与"副作用"）则判定为已覆盖。

## 5. FacetGraph-QA Agent（核心回答生成器）[最复杂]

### 输出格式决策 [手动填写]
- 思考部分：&lt;think&gt;\n&lt;facet=XXX&gt;\n思考过程\n&lt;/think&gt;
- 正文部分：无工具痕迹的完整回答

### 禁止词汇清单 [手动整理，严格管控]
绝对禁止出现：
- "查阅/检索/搜索"
- "工具返回/调用接口"
- "从图谱里查到"
- "我查了一下/系统显示"
- "根据检索结果"
- "retrieved/searched/looked up/tool says"

允许使用（出处式表达）：
- "在《XX指南》第3章提到..."
- "某法规第X条明确规定..."
- "说明书【禁忌】部分写明..."

### 证据引用格式 [手动设计]
- refs证据：[证据R1:来源=refs:《文档名/记录名》]
- 图谱证据：[证据G1:来源=图谱:节点ID/关系路径]

## 6. Facet Redundancy Detector（冗余检测器）

### 输入 [手动明确]
- Q: string
- planners: array of {planner, answer}

### 输出 [手动明确]
- JSON数组：需要删除的下标（如[1,3]）
- 空数组表示无冗余

### 冗余判断标准 [手动设计]
视为冗余的情况：
1. 核心结论一致
2. 论证路径相同
3. 证据结构高度重叠
4. 只是措辞变化
5. 只是补充细节但逻辑骨架一致

视为不同的情况：
1. 回答组织逻辑不同
2. 强调重点不同
3. 结论生成路径不同
4. 关注维度不同

---

## 7. Multi-Answer Synthesis Agent（综合总结器）

### 输出格式决策 [手动填写]
- 非数组，单一整合文本
- 结构：
  1. 结论概览
  2. 核心依据整合
  3. 完整展开说明
  4. 风险与边界条件
  5. 实务/操作建议
  6. 不确定性说明（如适用）

### 冲突处理策略 [手动设计]
- 证据充分性优先
- 时效性优先
- 无法裁决时：条件化说明（在A情况下...；在B情况下...）

### 禁止事项 [手动整理]
- 不要输出数组
- 不要引用"回答1/回答2"
- 不要提及"模型说"
- 不要暴露分析过程
- 不要出现英文
